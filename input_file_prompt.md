# Промт для генерации файла конфигурации (JSON) для загрузки в SZI Optimizer

Текст ниже используется как промт для ИИ, чтобы получить корректный JSON-файл для кнопки **«Загрузить файл»**.

---

Сгенерируй JSON-файл конфигурации для приложения SZI Optimizer.

Требования к формату:
1. Верни **только валидный JSON** без комментариев и без markdown.
2. Поля JSON:
   - `assets` — целое число `m` (>= 1)
   - `protections` — целое число `n` (>= 1)
   - `lambda` — число в диапазоне `[0, 1]`
   - `defuzzMethod` — одно из значений: `"centroid"`, `"yager"`, `"graded_mean"`, `"mode"`
   - `cFuzzy` — массив длины `n`, каждый элемент массив из 3 чисел `[a,b,c]`
   - `dFuzzy` — массив длины `n`, каждый элемент массив из 3 чисел `[a,b,c]`
   - `costs` — матрица размера `m x n`
   - `budgets` — массив длины `m`
3. Во всех строках `cFuzzy` и `dFuzzy` должно выполняться `a <= b <= c`.
4. Все значения в `costs` и `budgets` должны быть неотрицательными.
5. Для каждой строки матрицы `costs` должно выполняться условие:
   `max(costs[i]) <= budgets[i]`.
6. Размерности должны строго совпадать с `assets` и `protections`.

Верни пример реалистичных чисел для теста.

---

## Пример структуры JSON

```json
{
  "assets": 3,
  "protections": 5,
  "lambda": 0.5,
  "defuzzMethod": "centroid",
  "cFuzzy": [[80, 90, 100], [70, 76, 82], [24, 30, 36], [30, 35, 41], [26, 30, 34]],
  "dFuzzy": [[30, 40, 50], [15, 20, 25], [10, 14, 18], [8, 12, 16], [6, 10, 14]],
  "costs": [[15, 10, 5, 4, 3], [27, 18, 12, 6, 6], [40, 25, 12, 11, 8]],
  "budgets": [20, 39, 48]
}
```
